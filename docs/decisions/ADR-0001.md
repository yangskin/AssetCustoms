# ADR-0001: 采用文档与工作流程规范

- 状态：Accepted
- 日期：2025-11-02

## 背景
在 Unreal 插件的 Python 工程中，早期缺少统一的文档与工程约束，导致协作成本高、知识沉淀不足。

## 决策
- 建立文档目录（`docs/`、`standards/`），并在 `README.md` 做统一索引。
- 采用 Conventional Commits 作为提交规范。
- 引入 Code Review 检查单与编码规范文件。
- 后续重要决策以 ADR 形式记录在 `docs/decisions/` 下。

## 影响
- 贡献者可以快速发现信息与规范，降低沟通成本。
- 初期需要少量维护成本，但长期收益明显。

## 备选方案
- 仅在 README 维护零散约定（放弃：难以扩展与追踪）。
- 直接引入完整 CI/模板体系（暂缓：先轻量落地，再渐进增强）。
